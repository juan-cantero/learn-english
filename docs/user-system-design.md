### Join Flow

```
1. Teacher creates classroom â†’ System generates unique code (e.g., "XK92")
2. Teacher shares code with students (WhatsApp, email, in-class)
3. Student enters code in app â†’ Joins classroom
4. Student appears in teacher's dashboard
```

### Assignment Flow

```
1. Teacher browses catalog or generates new episode
2. Teacher clicks "Assign to Classroom"
3. Selects classroom, sets due date, adds instructions (optional)
4. Students see assignment in their dashboard
5. Students complete exercises
6. Teacher sees real-time progress
```

## 4. Data Model

### Core Tables

```sql
-- =============================================
-- USERS & AUTHENTICATION
-- =============================================

-- Main user table (synced with Supabase Auth)
CREATE TABLE users (
    id UUID PRIMARY KEY,                    -- From Supabase Auth
    email VARCHAR(255) NOT NULL UNIQUE,
    display_name VARCHAR(100),
    avatar_url VARCHAR(500),
    role VARCHAR(20) DEFAULT 'LEARNER',     -- LEARNER, TEACHER
    preferred_difficulty VARCHAR(20),        -- BEGINNER, INTERMEDIATE, ADVANCED
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- =============================================
-- CLASSROOMS (Teacher Feature)
-- =============================================

CREATE TABLE classrooms (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    teacher_id UUID NOT NULL REFERENCES users(id),
    name VARCHAR(100) NOT NULL,
    description TEXT,
    join_code VARCHAR(10) NOT NULL UNIQUE,  -- Random code like "XK92"
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE classroom_students (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    classroom_id UUID NOT NULL REFERENCES classrooms(id) ON DELETE CASCADE,
    student_id UUID NOT NULL REFERENCES users(id),
    joined_at TIMESTAMP DEFAULT NOW(),
    is_active BOOLEAN DEFAULT TRUE,

    UNIQUE(classroom_id, student_id)
);

-- =============================================
-- ASSIGNMENTS
-- =============================================

CREATE TABLE assignments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    classroom_id UUID NOT NULL REFERENCES classrooms(id) ON DELETE CASCADE,
    episode_id UUID NOT NULL REFERENCES episodes(id),
    title VARCHAR(200),                      -- Optional custom title
    instructions TEXT,                       -- Teacher's notes
    due_date TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE assignment_submissions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    assignment_id UUID NOT NULL REFERENCES assignments(id) ON DELETE CASCADE,
    student_id UUID NOT NULL REFERENCES users(id),
    status VARCHAR(20) DEFAULT 'NOT_STARTED', -- NOT_STARTED, IN_PROGRESS, COMPLETED
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    score INTEGER,                           -- 0-100
    time_spent_minutes INTEGER,

    UNIQUE(assignment_id, student_id)
);

-- =============================================
-- USER PROGRESS (For free learning)
-- =============================================

CREATE TABLE user_episode_progress (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id),
    episode_id UUID NOT NULL REFERENCES episodes(id),
    vocabulary_score INTEGER DEFAULT 0,
    grammar_score INTEGER DEFAULT 0,
    expressions_score INTEGER DEFAULT 0,
    exercises_score INTEGER DEFAULT 0,
    exercises_completed JSONB DEFAULT '[]',  -- Array of exercise IDs
    completion_percentage INTEGER DEFAULT 0,
    last_activity TIMESTAMP DEFAULT NOW(),

    UNIQUE(user_id, episode_id)
);

-- =============================================
-- USER STATISTICS
-- =============================================

CREATE TABLE user_stats (
    user_id UUID PRIMARY KEY REFERENCES users(id),
    total_episodes_completed INTEGER DEFAULT 0,
    total_exercises_completed INTEGER DEFAULT 0,
    total_words_learned INTEGER DEFAULT 0,
    current_streak_days INTEGER DEFAULT 0,
    longest_streak_days INTEGER DEFAULT 0,
    last_activity_date DATE,
    updated_at TIMESTAMP DEFAULT NOW()
);

-- =============================================
-- INDEXES
-- =============================================

CREATE INDEX idx_classroom_teacher ON classrooms(teacher_id);
CREATE INDEX idx_classroom_code ON classrooms(join_code);
CREATE INDEX idx_classroom_students_student ON classroom_students(student_id);
CREATE INDEX idx_assignments_classroom ON assignments(classroom_id);
CREATE INDEX idx_submissions_student ON assignment_submissions(student_id);
CREATE INDEX idx_progress_user ON user_episode_progress(user_id);
CREATE INDEX idx_progress_episode ON user_episode_progress(episode_id);
```

### JWT Flow

```
1. User logs in via Supabase (frontend)
2. Supabase returns JWT token
3. Frontend stores token (localStorage/cookie)
4. Frontend sends token with every API request:
   Authorization: Bearer <jwt_token>
5. Spring Security validates JWT signature
6. Spring extracts user_id from JWT claims
7. API uses user_id for all operations
```

## 8. API Endpoints (New)

### Classrooms

```
POST   /api/v1/classrooms                    # Create classroom (teacher)
GET    /api/v1/classrooms                    # List my classrooms
GET    /api/v1/classrooms/{id}               # Get classroom details
PUT    /api/v1/classrooms/{id}               # Update classroom
DELETE /api/v1/classrooms/{id}               # Delete classroom
POST   /api/v1/classrooms/{id}/regenerate-code  # New join code

POST   /api/v1/classrooms/join               # Join with code (student)
DELETE /api/v1/classrooms/{id}/leave         # Leave classroom (student)
GET    /api/v1/classrooms/{id}/students      # List students (teacher)
DELETE /api/v1/classrooms/{id}/students/{studentId}  # Remove student
```

### Assignments

```
POST   /api/v1/classrooms/{id}/assignments   # Create assignment
GET    /api/v1/classrooms/{id}/assignments   # List assignments
GET    /api/v1/assignments/{id}              # Get assignment details
PUT    /api/v1/assignments/{id}              # Update assignment
DELETE /api/v1/assignments/{id}              # Delete assignment

GET    /api/v1/assignments/{id}/submissions  # List submissions (teacher)
GET    /api/v1/my/assignments                # My assignments (student)
POST   /api/v1/assignments/{id}/start        # Start assignment
POST   /api/v1/assignments/{id}/complete     # Complete assignment
```

### User Profile & Progress

```
GET    /api/v1/me                            # Get current user profile
PUT    /api/v1/me                            # Update profile
POST   /api/v1/me/upgrade-to-teacher         # Upgrade account

GET    /api/v1/me/progress                   # Overall progress
GET    /api/v1/me/progress/episodes/{id}     # Episode progress
GET    /api/v1/me/stats                      # Learning statistics
```
